# 나의 개발일지
> "나의 개발일지"는 개발자가 자신의 프로젝트 수행 이력을 **포토북 형태의 포트폴리오로 제작**할 수 있도록 도와주는 서비스입니다.
---

## 1. 서비스 소개
### 타겟 고객
- **신입 개발자 취업 준비생** — 개발 포트폴리오 제작 경험이 부족하거나, PPT 형식의 한계를 느끼는 개발자
- **경력 개발자** — 누적된 프로젝트 이력을 정리하여 차별화된 형태로 본인의 역량을 어필하고자 하는 개발자

### 주요 기능
**1. 프로젝트 편집** <br/>
표지 정보(이름, 직무, 활동 기간 등)와 프로젝트별 세부 내용(기간, 역할, 기술 스택, 주요 성과, 스크린샷)을 폼 형태로 입력

**2. 미리보기** <br/>
입력한 내용을 확인할 수 있도록 포토북 제작 전 미리보기 형태로 확인

**3. 포토북 주문** <br/>
배송지 정보 입력 후 Sweetbook API를 통해 실제 포토북 제작 및 주문 생성

**4. 실시간 제작 현황** <br/>
SSE(Server-Sent Events) 기반으로 포토북 생성 진행률을 실시간으로 화면에 렌더링

**5. 카탈로그 조회** <br/>
Sweetbook이 제공하는 포토북 판형과 테마별로 그룹화된 포토북 템플릿 목록 조회
<br/>

## 2. 실행 방법
* 이 프로젝트는 frontend와 backend가 분리된 구조이며, **백엔드를 먼저 실행**한 후 프론트엔드를 실행해야 합니다.
* 아래 명령어는 모두 git clone 후 **프로젝트 루트에서 시작**하는 기준으로 작성되었습니다.

### 버전
* Node.js v22.13.0 이상 버전이 필요합니다. (개발 환경: Node.js v22.15.0, OS: Windows)
* [Sweetbook Partner Portal](https://api.sweetbook.com/)에서 발급받은 API Key가 필요합니다. (Sandbox 환경)

### 백엔드 실행
``` 
cd backend
npm install
cp .env.example .env

# .env 파일을 열어 SWEETBOOK_API_KEY 값을 본인의 키로 교체

npm run dev
```
<br/>

* 정상 실행 시 터미널에 다음과 같이 표시됩니다.
```
> backend@1.0.0 dev
> tsx watch src/index.ts

◇ injected env (4) from .env // tip: ⌘ multiple files { path: ['.env.local', '.env'] }
Environment variables loaded
🚀 Server running on http://localhost:3000
   Environment: sandbox
```

### 프론트엔드 실행
* 프로젝트 루트에서 **새 터미널**을 열고 아래 명령어를 실행합니다.
```
cd frontend
npm install
cp .env.example .env
npm run dev
```
* 프론트엔드 프로젝트에도 **.env 파일 생성이 필요**합니다.
* `VITE_API_BASE_URL`은 API 요청의 기본 주소로 사용되는 필수 변수이며, 백엔드를 기본 포트(3000)로 실행한 경우 `.env.example`의 기본값을 그대로 사용하시면 됩니다.
* 정상 실행 시 [http://localhost:5173](http://localhost:5173)에서 개발 서버가 구동되며, 접속 시 바로 서비스를 확인할 수 있습니다.
* 반드시 **[http://localhost:5173](http://localhost:5173)** 주소로 접속해야 합니다. (IP 주소로 접속 시 API 통신 문제가 발생합니다.)
<br/>

## 3. 사용한 API 목록
1권의 포토북을 생성하고 주문하기까지 다음 5개의 API 엔드포인트를 순차 호출합니다.
### 포토북 생성 및 주문
|API|용도|
|:---:|:---:|
|**POST /books**|책 생성|
|**POST /books/{bookUid}/cover**|책 표지 추가|
|**POST /books/{bookUid}/contents**|책 콘텐츠 추가 (60회 반복)|
|**POST /books/{bookUid}/finalization**|책 최종화|
|**POST /orders**|주문 생성|

### 카탈로그 조회
판형 및 템플릿 카탈로그 페이지 구현을 위해 다음의 조회 API를 사용했습니다.
|API|용도|
|:---:|:---:|
|**GET /book-specs**|판형 조회|
|**GET /templates**|템플릿 목록 조회|

## 4. AI 도구 사용 내역
|AI 도구|활용 내용|
|:---|:---|
|**Claude Code**|백엔드(Node.js, Express) 개념 학습, 외부 API 응답 구조 분석, 에러 처리 설계 논의 등 문제 해결 보조|
|**ChatGPT**|실제 수행한 프로젝트 데이터 4개의 형식을 토대로 가상 프로젝트 26개의 Mockup 데이터 생성|
|**GitHub Copilot**|일부 SCSS 스타일 수정(ex. 상위 블록 스타일을 고려한 여백 조정 등)을 위한 보조 용도|
|**Gemini**|나노바나나2 모델로 로고 이미지 생성|

### 활용 내용 상세
**1. 백엔드 기술 개념 이해**
* 백엔드 개발이 처음인 상황에서 프로젝트 세팅, 생소한 개념 정리, API 응답 구조 파악, 에러 처리 설계를 위해 Claude Code를 활용했습니다.

**2. 외부 API 응답 구조 분석**
* Sweetbook API 응답 구조 파악을 위해 응답값을 Claude Code와 함께 json 데이터로 저장하고, 에디터 포맷팅 단축키를 사용했습니다.
* 이를 통해 템플릿마다 응답 데이터 형태가 다름을 직접 파악하여 "개발 일지" 컨셉에 맞는 판형과 템플릿을 선택했습니다.

**3. 에러 처리 전략 논의**
* SSE 스트림(포토북 제작) 도중 서버 연결이 끊어졌을 때와 서버가 꺼져있어 발생하는 에러 메시지가 동일하게 표시되는 문제를 발견했습니다.
* 이는 사용자가 재주문을 시도할 경우, 이중 결제로 이어질 수 있는 중요한 문제라고 판단하여 Claude Code와 해결 방안에 대해 논의했습니다.
* `hasStartedStreaming` 플래그로 상태를 관리하며 에러 분류, 메시지 표시, UI 상태 각각의 책임을 분리하도록 직접 설계하고 구현했습니다.
* 이를 통해 스트림 도중 연결이 끊어지면 '다시 주문하지 마시고 고객센터에 문의해주세요.'라는 안내 UI를 띄우도록 하였습니다.

**4. 가상 프로젝트 26개 Mockup 데이터 생성**
* 포토북 생성을 위한 최소 페이지 조건(50페이지 이상)을 맞추기 위해 가상 프로젝트 26개의 데이터를 ChatGPT로 생성했습니다.
* `mockPortfolio.ts`의 처음 4개 프로젝트가 실제로 수행한 프로젝트이며, 나머지는 가상 프로젝트 데이터입니다.

**5. GitHub Copilot으로 스타일 수정 보조**
* 부모 요소를 고려한 스타일 수정, 공통 컴포넌트의 일부 스타일 수정 시 IDE 내에서 맥락을 파악하여 수정하도록 사용했습니다.
* 이를 통해 기존 스타일을 바꾸지 않으면서도 의도한 스타일이 설정되도록 하였습니다.

## 5. 설계 의도
### 왜 이 서비스를 선택했는가
이 서비스는 **개발자로서 직접 포트폴리오를 만드는 과정에서 느낀 불편함**에서 출발했습니다.

취업 준비 과정에서 주변 친구들이 개발 포트폴리오를 PPT로 많이 만들고 있었고, 저의 첫 포트폴리오 또한 PPT로 제작했습니다. <br/>
그러나 PPT 형식으로 개발 포트폴리오를 만들며 3가지 한계를 지속적으로 마주했습니다.

**1. 이미지와 설명 사이의 딜레마**
* 화면 UI나 코드 스크린샷을 넣으면 설명을 쓸 공간이 좁아졌습니다.
* 반대로 설명을 충분히 넣으면 슬라이드의 가독성이 떨어졌습니다.

**2. GIF의 제약**
* 동작 과정을 GIF로 담으면 시각 효과가 돋보였지만 파일 용량이 커졌습니다.
* 상당수 기업들이 PDF 형식의 포트폴리오 제출을 요구하는 상황에서, PDF 제출 시 gif 애니메이션이 동작하지 않아 효과가 무색해졌습니다.

**3. 웹 사이트 형식의 진입 장벽**
포트폴리오 웹 사이트 제작 도중 디자인 시스템 구축에 어려움을 느껴 중도 포기하는 모습도 보았습니다.

이런 경험을 바탕으로 프로젝트 정보와 사진, 설명을 입력하면 **알아서 포토북 형태로 제작**되는 구조를 만들고 싶었습니다. 이는 디자인에 대한 개발자들의 부담을 줄이면서도, **포트폴리오 정보와 가독성의 균형을 유지**한 질 좋은 결과물을 제공할 수 있기 때문입니다.

최종 결과물인 실물 포토북을 면접장까지 들고 가는 경우는 거의 없겠지만, 실물 제작 단계에서 생성되는 PDF 형태 결과물만으로 **PPT보다 효율적인 어필 수단**이 될 수 있다고 판단했습니다. 포토북이라는 매체의 **디자인 완성도**와 취업 시장에서 실제로 사용할 수 있는 **실용성**을 챙길 수 있기 때문입니다. <br/>

### 비즈니스 가능성
취업 포털 사이트에는 자신의 이력서가 잘 노출되도록 하는 프리미엄 서비스가 운영되고 있습니다. 이는 '취업 준비' 영역에서 일정 금액을 지불할 의사가 있는 사용자층이 존재함을 의미합니다. <br/>

이를 토대로 본 서비스는 **신입 개발자 취업 준비생**과 **경력 이직 준비 개발자**를 고객으로 삼을 수 있습니다. 그리고 실물 포토북을 주문한 고객과 포토북의 PDF 파일만 주문한 고객의 가격 차이를 둠으로써, 포토북 제작 비용을 고려하여 서비스 가격을 책정할 수 있습니다.

### 시간이 더 있었다면 추가했을 기능
**1. 사진 직접 첨부** <br/>
* 현재는 사용자가 첨부할 이미지의 URL을 직접 입력하는 방식입니다.
* 한편 사용자는 자신의 PC나 모바일에서 이미지 파일을 직접 업로드하는 UX를 기대할 것입니다.
* 이를 구현하려면 업로드한 이미지를 스토리지에 저장하고, 반환받은 URL을 Sweetbook API로 전달하는 과정이 필요합니다.
* 본 과제에서는 제한된 시간 내에 포토북을 제작해야 했으므로, 이미지 URL 입력 방식으로 과정을 축소하여 핵심 플로우 구현에 집중했습니다.

**2. 로그인 및 회원 관리** <br/>
* "나의 개발일지" 서비스는 로그인 없는 단일 세션 데모로 구동됩니다.
* 하지만 보다 수준높은 서비스가 되기 위해서는 회원 관리를 통한 주문 내역 조회, 주문 취소 및 환불 등의 기능이 필요합니다.
* Orders API의 주문 목록 조회, 주문 취소 메서드를 이용하여 추가 구현이 가능할 것으로 예상됩니다.

**3. 카탈로그 페이지와 편집 페이지의 UX 흐름 연동** <br/>
* 현재 판형 목록과 템플릿 목록 페이지는 조회 전용으로, 썸네일 이미지를 클릭할 수 없습니다.
* 본 과제에서는 포토북 생성, 주문 플로우의 안정성을 유지하기 위해 카탈로그 페이지를 조회 전용 페이지로 구현했습니다.
* 추후 템플릿 파라미터 조정, Mapper 레이어 재설계 등으로 카탈로그 조회 과정과 프로젝트 편집 과정이 하나의 UX 흐름으로 연동될 수 있습니다.
